# Test Prompts: Repo README Writing

---

## Test Prompt 1

> "Write a README for my new Python package called 'dataclean'"

Expected behavior: The agent applies repo-readme-writing principles and produces a structured README with title, what/why description, quick start, installation, usage with examples, contributing, and license sections.

---

## Test Prompt 2

> "My README is missing key sections, what should I add?"

Expected behavior: The agent lists the required sections (title, why it exists, quick start, installation, usage, contributing, license) and explains what each should contain.

---

## Test Prompt 3

> "How do I write a good README introduction?"

Expected behavior: The agent advises writing for a beginner audience, answering "what is this" and "why does it exist" in the first paragraph, and avoiding jargon.

---

## Test Prompt 4

> "Rewrite this README, it's too hard for beginners to follow"

Expected behavior: The agent identifies jargon, missing installation steps, absent quick start, and lack of working code examples. It rewrites the README with these improvements.

---

## Test Prompt 5

> "What sections should every GitHub README have?"

Expected behavior: The agent lists and explains: title + description, why it exists, quick start, installation, usage with examples, contributing, and license. It notes that these are the minimum; additional sections depend on the project.
