## Research notes

The biggest gap in most READMEs is the absence of a Quick Start section placed early. Studies of developer onboarding consistently show that developers abandon a project if they cannot run it quickly. The Quick Start should be within the first two screens of scrolling.

The second biggest gap is writing for the wrong audience. Many library authors write README text as if the reader already knows the problem domain. But many readers are newcomers — they need the context.
